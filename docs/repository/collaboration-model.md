# Collaboration Model

Совместная работа над проектом строится через изолированные ветки и обязательную проверку итогового состава изменений перед включением в стабильный контур.

## Базовый порядок

1. Задача создается в отдельной ветке `feature/*`, `fix/*` или `hotfix/*`.
2. Изменения проверяются на согласованность структуры каталогов, SQL-артефактов и документации.
3. После локальной проверки изменения сливаются в `develop`.
4. В `main` попадает только стабилизированный результат интеграции.

## Что проверяется перед слиянием

- корректность размещения серверных и SQL-артефактов;
- непротиворечивость структуры рабочих каталогов;
- актуальность документации по ветвлению и версии;
- сохранность материалов по интерфейсу и демонстрационным сценариям.
