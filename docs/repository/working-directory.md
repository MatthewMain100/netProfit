# Working Directory

Рабочая директория проекта организована по модульному принципу, чтобы серверный контур, база данных, документация и клиентские материалы сопровождались независимо.

## Каталоги верхнего уровня

- `backend/` - серверные сценарии и SQL-артефакты, необходимые для расчетного контура.
- `frontend/` - клиентский контур и материалы, связанные с пользовательским интерфейсом.
- `database/` - отдельный набор восстановительных SQL-снимков.
- `docs/` - регламенты, скриншоты и сопроводительная проектная документация.

## Правила размещения

- миграции хранятся только в `backend/db/migrations/`;
- базовая схема хранится только в `backend/db/schema/`;
- стартовые данные и демонстрационные наборы размещаются в `backend/db/seeds/`;
- тяжелые SQL-снимки не дублируются в рабочих каталогах и хранятся в `database/restores/`;
- скриншоты интерфейса и иные визуальные материалы размещаются в `docs/screenshots/`.

## Практический смысл

Такое разделение снижает связанность между компонентами и позволяет отдельно сопровождать:

- серверную бизнес-логику;
- данные и восстановительные сценарии;
- визуальные материалы и проектную документацию.
