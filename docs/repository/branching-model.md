# Branching Model

В репозитории используется двухуровневая схема сопровождения с выделением стабильного и интеграционного контуров.

## Основные ветки

- `main` - стабильная ветка для демонстрации и фиксации проверенного состояния.
- `develop` - интеграционная ветка для объединения завершенных задач перед выпуском в `main`.

## Специализированные ветки

- `feature/*` - реализация функциональных задач и структурных доработок.
- `fix/*` - исправление ошибок, несоответствий документации и регламентов.
- `hotfix/*` - срочные правки стабильного контура после публикации в `main`.

## Правила

- прямые изменения в `main` не допускаются;
- завершенные `feature/*` и `fix/*` ветки сначала интегрируются в `develop`;
- `hotfix/*` создаются от `main` и после проверки возвращаются в `main` и `develop`;
- merge-коммиты сохраняются для восстановления этапности разработки.
