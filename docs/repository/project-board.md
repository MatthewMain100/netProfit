# Project Board Workflow

GitHub Project используется как внешний контур контроля задач по репозиторию.

## Статусы

- `Backlog` - задача зафиксирована, но работа не начата.
- `In Progress` - ведется работа в отдельной ветке.
- `Review` - изменения собраны и ожидают проверки через Pull Request.
- `Done` - задача завершена и интегрирована в целевую ветку.

## Привязка к веткам

- задачи развития оформляются в ветках `feature/*`;
- задачи исправления оформляются в ветках `fix/*`;
- срочные релизные корректировки оформляются в ветках `hotfix/*`.

## Поток выполнения

1. Issue фиксируется в Project и получает статус `Backlog`.
2. После создания рабочей ветки задача переводится в `In Progress`.
3. При открытии Pull Request задача переводится в `Review`.
4. После merge задача переводится в `Done`.
