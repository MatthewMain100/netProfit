# Versioning Model

Для проекта используется схема `MAJOR.MINOR.PATCH`.

## Интерпретация версий

- `MAJOR` - крупное изменение архитектурного или предметного контура;
- `MINOR` - расширение функциональности без нарушения совместимости;
- `PATCH` - локальные исправления и выравнивание стабильного контура.

## Применение в репозитории

- версия `2.0.0` фиксирует текущий расширенный контур платформы Net Profit 2.0;
- последующие исправления стабильной ветки публикуются как `2.0.x`;
- накопленные функциональные расширения в `develop` поднимают номер `MINOR`.
